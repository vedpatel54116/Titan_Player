# App Store Preparation & Dual Distribution Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship TitanPlayer through both the Mac App Store and notarized Developer-ID direct distribution by producing the entitlements files, asset catalog, fastlane metadata, dual-configuration Xcode project, license, and privacy document — with a textual lint check (`make dist-prep-check`) that can run in this CommandLineTools-only environment.

**Architecture:** A `project.yml` is the single source of truth for the Xcode build; `xcodegen generate` produces `TitanPlayer.xcodeproj/project.pbxproj`, which is gitignored. Two Xcode configurations (`AppStore`, `Direct`) build the same SwiftPM-produced executable but apply different entitlements, signing identities, and sandbox flags. `fastlane` keeps MAS and Direct lanes scriptable. Text-only verification runs in CI and locally without Xcode.

**Tech Stack:** xcodegen, fastlane, xcodeproj-format pbxproj, plistlint, xmllint, ruby json/yaml. No Swift source changes.

**Honest env limit:** This machine has CommandLineTools only. Steps that require `xcodebuild`, `xcrun notarytool`, `xcrun altool`, `iconutil`, or `fastlane` are documented as "user-side" — the textual lint gates run here. Every artifact this prompt produces is structurally validated before commit.

```markdown
# TitanPlayer Privacy Policy

_Last updated: 2026-06-30_

TitanPlayer is an open-source macOS video player. This page describes what
data the application handles, why, and where it goes.

## Data TitanPlayer Handles

| Data | Where it lives | Why |
|---|---|---|
| Media files you open (videos, audio, sidecar subtitle/cue files) | Local on your Mac | Playback. Never transmitted off-device unless you explicitly use a network-streaming feature. |
| Recent-played file list | Local on your Mac (`~/Library/Application Support/TitanPlayer/Recent.plist`) | Resume your library across launches. |
| Window positions, mini-player size | Local on your Mac (`~/Library/Preferences/com.titanplayer.app.plist`) | Restore your layout. |
| Crash logs (only if you enable them) | Local on your Mac | Optional debugging; off by default. |
| Crash reports you choose to send | Off-device only if you submit them yourself | You control submission; TitanPlayer has no auto-upload. |

## Network Uses

TitanPlayer can connect to the network **only when you enable a streaming
source yourself** — for example an HTTP Live Stream (HLS) URL, a remote
subtitle provider, or an SMB/NFS share. TitanPlayer does not perform
background telemetry, beaconing, or analytics of any kind.

When the App Sandbox is enabled (Mac App Store build):
- Outgoing TCP/UDP (e.g. streaming) is allowed for `com.apple.security.network.client`.
- Incoming TCP/UDP (e.g. AirPlay-server mode on the local network) is allowed for `com.apple.security.network.server`.
- Local network mDNS/Bonjour is permitted by NSAllowsLocalNetworking = YES.

## Microphone

The microphone is read **only** when you explicitly enable audio capture /
voice-over features. Audio is processed locally; TitanPlayer does not record,
store, or transmit microphone audio.

## Apple Events

TitanPlayer may send Apple Events to hand off media to QuickTime or other
players, and to receive Remote control events. These never leave your Mac.

## Open-Source Code

TitanPlayer's source is available under the MIT License. Auditing the
network code is encouraged; the file paths are:

- `TitanPlayer/TitanPlayer/Streaming/`
- `TitanPlayer/TitanPlayer/Networking/`

## Children's Privacy

TitanPlayer does not target children and does not knowingly collect any
personal data from any user.

## Changes to This Policy

Material changes are documented in the git log of this file. Open-source
consumers are encouraged to PR improvements.

## Contact

Privacy questions: open an issue at the project repository.
```

```swift
#!/usr/bin/env swift
// scripts/render-placeholder-icon.swift
// Generates a clean baseline placeholder AppIcon for TitanPlayer.
//
// Output: TitanPlayer/TitanPlayer/Resources/Assets.xcassets/AppIcon.appiconset/icon_<size>.png
//   plus  TitanPlayer/TitanPlayer/Resources/Icon-Placeholder/master.png  (1024×1024)
//
// Pure-Swift on macOS. Uses Core Graphics only. No third-party deps.

import Foundation
import CoreGraphics
import ImageIO
import UniformTypeIdentifiers

@discardableResult
func writePNG(_ url: URL, _ image: CGImage) -> Bool {
    let dest = CGImageDestinationCreateWithURL(url as CFURL, UTType.png.identifier as CFString, 1, nil)!
    CGImageDestinationAddImage(dest, image, nil)
    return CGImageDestinationFinalize(dest)
}

func renderIcon(size: Int) -> CGImage {
    let cs = CGColorSpaceCreateDeviceRGB()
    let ctx = CGContext(data: nil, width: size, height: size,
                        bitsPerComponent: 8, bytesPerRow: 0, space: cs,
                        bitmapInfo: CGImageAlphaInfo.premultipliedLast.rawValue)!
    let s = CGFloat(size)

    // Rounded square mask (macOS-style squircle approximation)
    let path = CGPath(roundedRect: CGRect(x: 0, y: 0, width: s, height: s),
                      cornerWidth: s * 0.225, cornerHeight: s * 0.225,
                      transform: nil)

    // Background gradient (deep blue → near-black)
    let gradient = CGGradient(colorsSpace: cs,
                              colors: [
                                CGColor(red: 0.10, green: 0.20, blue: 0.55, alpha: 1),
                                CGColor(red: 0.04, green: 0.08, blue: 0.22, alpha: 1)
                              ] as CFArray,
                              locations: [0, 1])!
    ctx.addPath(path)
    ctx.clip()
    ctx.drawLinearGradient(gradient,
                           start: .zero,
                           end: CGPoint(x: 0, y: s),
                           options: [])

    // White play triangle, centered.
    let triW = s * 0.36
    let cx = s / 2
    let cy = s / 2
    let triPath = CGMutablePath()
    triPath.move(to: CGPoint(x: cx - triW * 0.5, y: cy - triW * 0.6))
    triPath.addLine(to: CGPoint(x: cx + triW * 0.5, y: cy))
    triPath.addLine(to: CGPoint(x: cx - triW * 0.5, y: cy + triW * 0.6))
    triPath.closeSubpath()

    ctx.setFillColor(CGColor(red: 1, green: 1, blue: 1, alpha: 0.94))
    ctx.addPath(triPath)
    ctx.fillPath()

    return ctx.makeImage()!
}

let args = CommandLine.arguments
guard args.count >= 3 else {
    print("usage: render-placeholder-icon.swift <out-dir> <sizes...>")
    exit(2)
}
let outDir = URL(fileURLWithPath: args[1], isDirectory: true)
try? FileManager.default.createDirectory(at: outDir, withIntermediateDirectories: true)

var sizes: [(Int, String)] = []
var i = 2
while i < args.count {
    let s = Int(args[i])!
    let name: String
    switch s {
    case 16:        name = "icon_16x16.png"
    case 32:        name = "icon_16x16@2x.png"
    case 64:        name = "icon_32x32@2x.png"
    case 128:       name = "icon_128x128.png"
    case 256:       name = "icon_128x128@2x.png"
    case 512:       name = "icon_256x256@2x.png"
    case 1024:      name = "icon_512x512@2x.png"
    default:
        print("skip unmatched size \(s)")
        i += 1
        continue
    }
    sizes.append((s, name))
    i += 1
}

let master = renderIcon(size: 1024)
let masterURL = outDir.appendingPathComponent("../Icon-Placeholder/master.png")
try? FileManager.default.createDirectory(at: masterURL.deletingLastPathComponent(),
                                         withIntermediateDirectories: true)
_ = writePNG(masterURL, master)

for (s, name) in sizes {
    let img = renderIcon(size: s)
    let url = outDir.appendingPathComponent(name)
    _ = writePNG(url, img)
    print("wrote \(url.path)")
}
```
